# ESP32-CAM 监控客户端

一个基于 Flutter 开发的跨平台 ESP32-CAM 视频监控客户端应用，支持实时查看摄像头画面和拍照保存功能。

## 功能特性

- ✅ **高清视频流**：支持 JPEG 实时轮询模式，稳定流畅
- ✅ **实时帧率显示**：显示当前视频流的 FPS
- ✅ **LED 闪光灯控制**（💡 新增）：
  - 📊 0-100% 亮度无级调节
  - 💡 自动映射到 ESP32 的 0-255 LED 强度
  - 🌙 显著改善低光环境下的画面质量
- ✅ **超全面的摄像头控制**（30+ 参数可调）：
  
  #### 📷 基础图像设置
  - 📐 分辨率调节（13档：QQVGA ~ UXGA）
  - 🎨 JPEG 压缩质量（0-63）
  - 💡 亮度（-2 ~ +2）
  - 🔲 对比度（-2 ~ +2）
  - 🌈 饱和度（-2 ~ +2）
  
  #### 🎨 高级图像处理
  - 📈 增益上限（7档：x2 ~ x128）
  - 🎭 特效（7种：负片、灰度、红/绿/蓝色调、复古）
  - ⚪ 白平衡模式（5种：自动、阳光、阴天、办公室、家庭）
  - 📊 曝光等级（-2 ~ +2）
  - ⏱️ AEC 值（0-1200，精确曝光控制）
  
  #### ⚙️ 自动控制系统
  - 🔄 自动白平衡 (AWB)
  - 📈 AWB 增益控制
  - 📊 自动增益 (AGC)
  - 📈 AGC 增益（0-30级）
  - ⏱️ 自动曝光 (AEC)
  - ⏱️ AEC2 (DSP加速)
  - 📉 DCW 下采样
  
  #### 🔧 图像校正
  - 🔴 坏点校正 (BPC)
  - ⚪ 白点校正 (WPC)
  - 🎨 Raw Gamma 校正
  - 📷 镜头畸变校正 (LENC)
  
  #### 🔄 图像变换
  - ↔️ 水平镜像
  - ↕️ 垂直翻转
  - 🎭 色彩条测试模式
  
- ✅ **拍照保存**：支持将当前画面保存到本地
  - 桌面端（Windows/Linux/macOS）：保存到当前运行目录
  - 移动端（Android/iOS）：保存到系统相册
- ✅ **跨平台支持**：支持 Android、iOS、Windows、Linux、macOS
- ✅ **分类清晰的设置面板**：5个功能区域，方便查找和调节
- ✅ **实时反馈**：所有调节立即生效，带操作提示

## 使用说明

### ESP32-CAM 服务端配置

确保您的 ESP32-CAM 已烧录 CameraWebServer 固件，并记录以下信息：

1. **IP 地址**：ESP32-CAM 的 IP 地址
   - AP 模式（热点）：通常是 `192.168.4.1`
   - STA 模式（连接到路由器）：在串口监视器中查看分配的 IP

2. **端口号**：
   - **JPEG 模式**：使用主端口（默认 `80`）
   - **MJPEG 流模式**：使用主端口 + 1（默认 `81`）

### 客户端配置步骤

1. **输入连接信息**：
   - IP 地址：输入 ESP32-CAM 的 IP 地址
   - 端口：输入主端口号（通常为 80）

2. **开始监控**：
   - 点击"开始"按钮
   - 观察状态栏显示连接状态和帧率
   - 使用 JPEG 轮询模式，稳定可靠

3. **摄像头设置**（⚙️ 点击右上角图标）：
   
   #### 💡 LED 闪光灯控制（新增）
   - **LED 亮度**：0-100% 无级调节
   - **自动映射**：百分比自动转换为 ESP32 的 0-255 强度值
   - **使用场景**：
     - 🌙 夜间监控：提高亮度照亮场景
     - 🌓 逆光补偿：补充前景光线
     - 🔦 手电筒模式：临时照明
   - **注意事项**：
     - 需要 ESP32 硬件支持 LED 引脚
     - 长时间高亮度可能导致发热
     - 建议根据环境光线动态调节
   
   #### 📐 图像参数调节
   弹出可拖拽的设置面板，包含以下控制项：
     - **分辨率**：从 QQVGA (160x120) 到 UXGA (1600x1200) 共13档
     - **图像质量**：0-63，数值越小质量越高
     - **亮度**：-2（暗）到 +2（亮）
     - **对比度**：-2（低）到 +2（高）
     - **饱和度**：-2（淡）到 +2（艳）
     - **自动白平衡**：开启/关闭
     - **自动增益**：开启/关闭
     - **自动曝光**：开启/关闭
     - **水平镜像**：开启/关闭
     - **垂直翻转**：开启/关闭
     - **色彩条测试**：开启/关闭（显示测试图案）
   - 调节滑块或开关后，设置会立即生效
   - 点击"刷新状态"可同步当前摄像头参数

4. **拍照保存**：
   - 点击"拍照"按钮或直接点击画面
   - 桌面端：保存到当前运行目录
   - 移动端：保存到系统相册

### 性能优化建议

#### 提高流畅度

1. **降低分辨率**：使用 QVGA (320x240) 或 VGA (640x480)
2. **提高图像质量值**：设置为 15-20（数值越大压缩越多，传输越快）
3. **确保良好的 WiFi 信号**：靠近路由器或 ESP32
4. **使用 LED 补光**：在低光环境下开启LED可提高帧率

#### 提高画质

1. **提高分辨率**：使用 SVGA (800x600) 或更高
2. **降低图像质量值**：设置为 5-10（数值越小画质越好）
3. **调节亮度和对比度**：根据环境光线调整
4. **开启自动白平衡**：获得更准确的颜色
5. **合理使用 LED**：避免过曝，适度补光

#### 低光环境优化

1. **开启 LED 闪光灯**：从 30% 开始逐步增加
2. **提高亮度设置**：+1 或 +2
3. **开启自动增益**：让摄像头自动增强信号
4. **降低分辨率**：较小的分辨率在低光下噪点更少

#### 降低延迟

1. **使用 MJPEG 模式**：实时性更好
2. **降低分辨率**：较小的图片传输更快
3. **减少网络拥塞**：避免其他设备占用带宽

### 常见问题排查

#### 1. 404 错误

**原因**：端点路径不正确或端口错误

**解决方案**：
- JPEG 模式确认使用 `/capture` 端点
- MJPEG 模式确认流服务器端口 = 主端口 + 1
- 在浏览器中测试：
  ```
  http://192.168.4.1/capture      # 应该返回一张图片
  http://192.168.4.1:81/stream    # 应该显示视频流
  ```

#### 2. 连接超时

**原因**：IP 地址错误或设备未启动

**解决方案**：
- 确认 ESP32-CAM 已上电并启动
- 检查设备是否连接到正确的 WiFi
- 尝试 ping ESP32 的 IP 地址
- 查看 ESP32 串口输出确认 IP 地址

#### 3. MJPEG 流无法连接

**原因**：流服务器端口配置错误

**解决方案**：
- 确认流服务器端口 = 主端口 + 1
- 例如：如果主端口是 80，流端口应该是 81
- 在浏览器中访问 `http://IP:81/stream` 测试

#### 4. 画面卡顿

**解决方案**：
- 切换到 MJPEG 模式（通常更流畅）
- 降低 ESP32 端的图像分辨率
- 降低 JPEG 质量设置
- 确保 WiFi 信号强度良好

## 技术实现

### 视频流处理

- **JPEG 模式**：使用 `http` 库定时请求 `/cam-hi.jpg` 端点
- **MJPEG 模式**：使用 `HttpClient` 建立持久连接到 `/stream` 端点，解析 multipart 数据流

### 权限管理

- 使用 `permission_handler` 库请求存储权限（仅移动端需要）
- 桌面端无需权限即可保存文件

### 图片保存

- 使用 `image_gallery_saver_plus` 库保存图片到系统相册（移动端）
- 桌面端直接使用 `dart:io` 的 `File` API 保存

## 常见问题

- 确认已授予存储权限
- 检查相册应用是否有写入权限
- 尝试重启应用

## 摄像头控制参数详解

### 💡 LED 闪光灯控制

| 参数 | 范围 | 说明 | ESP32映射 |
|------|------|------|----------|
| **LED 亮度** | 0-100% | 用户友好的百分比控制 | 自动转换为 0-255 |

**使用技巧：**
- 🌙 **夜间模式**：设置 50-80% 提供足够照明
- 🌓 **补光模式**：设置 20-40% 轻微补光
- 🔦 **强光模式**：设置 80-100% 最大亮度
- ⚠️ **注意**：长时间100%亮度可能导致LED发热

### 📷 基础图像设置

| 参数 | 范围 | 说明 | 推荐值 |
|------|------|------|--------|
| **分辨率** | 0-12 | 13档可选，QQVGA(160x120) ~ UXGA(1600x1200) | VGA(7) |
| **JPEG质量** | 0-63 | 压缩质量，越小越好 | 10-15 |
| **亮度** | -2 ~ +2 | 画面明暗程度 | 0 |
| **对比度** | -2 ~ +2 | 色彩对比强度 | 0 |
| **饱和度** | -2 ~ +2 | 色彩鲜艳程度 | 0 |

### 🎨 高级图像处理

| 参数 | 范围/选项 | 说明 | 使用场景 |
|------|----------|------|----------|
| **增益上限** | x2, x4, x8, x16, x32, x64, x128 | 信号放大倍数 | 低光环境提高灵敏度 |
| **特效** | 无、负片、灰度、红/绿/蓝色调、复古 | 特殊视觉效果 | 创意拍摄、调试 |
| **白平衡模式** | 自动、阳光、阴天、办公室、家庭 | 色温校正 | 不同光源环境 |
| **曝光等级** | -2 ~ +2 | 曝光补偿 | 逆光或过亮场景 |
| **AEC값** | 0-1200 | 精确曝光时间控制 | 专业曝光调节 |

### ⚙️ 自动控制系统

| 参数 | 类型 | 说明 | 建议 |
|------|------|------|------|
| **自动白平衡 (AWB)** | 开关 | 自动校正颜色偏差 | ✅ 开启 |
| **AWB增益** | 开关 | 白平衡信号增强 | ✅ 开启 |
| **自动增益 (AGC)** | 开关 | 自动调节信号增益 | ✅ 开启 |
| **AGC增益** | 0-30 | 增益强度级别 | 10-15 |
| **自动曝光 (AEC)** | 开关 | 自动调节曝光时间 | ✅ 开启 |
| **AEC2 (DSP)** | 开关 | 硬件加速曝光 | ✅ 开启（更快） |
| **DCW下采样** | 开关 | 数字下采样 | ✅ 开启 |

### 🔧 图像校正

| 参数 | 类型 | 说明 | 效果 |
|------|------|------|------|
| **坏点校正 (BPC)** | 开关 | 修正传感器坏点 | 减少噪点 |
| **白点校正 (WPC)** | 开关 | 修正过曝像素 | 改善高光细节 |
| **Raw Gamma** | 开关 | Gamma曲线校正 | 更自然的亮度响应 |
| **镜头校正 (LENC)** | 开关 | 畸变和暗角校正 | 边缘画质提升 |

### 🔄 图像变换

| 参数 | 类型 | 说明 | 使用场景 |
|------|------|------|----------|
| **水平镜像** | 开关 | 左右翻转画面 | 前置摄像头模式 |
| **垂直翻转** | 开关 | 上下翻转画面 | 倒装摄像头 |
| **色彩条测试** | 开关 | 显示彩色测试条 | 调试用 |

### 🎯 参数调节建议

#### 提高画面清晰度
1. **开启镜头校正 (LENC)**：修正边缘畸变
2. **开启坏点/白点校正**：减少噪点
3. **适当提高对比度**：+1 让画面更清晰
4. **选择合适的分辨率**：不要过高，VGA通常最佳

#### 改善低光表现
1. **提高增益上限**：x32 或 x64
2. **开启 AGC 并提高增益**：15-20
3. **开启 LED 补光**：50-70%
4. **提高亮度**：+1 或 +2
5. **降低分辨率**：QVGA 或 VGA

#### 优化色彩准确度
1. **开启 AWB**：自动白平衡
2. **选择合适的 WB 模式**：根据光源
3. **开启 Raw Gamma**：自然亮度响应
4. **饱和度保持 0**：避免过饱和

#### 减少延迟提高帧率
1. **降低分辨率**：最有效的方法
2. **提高 JPEG 质量值**：15-20
3. **关闭不必要的校正**：如 LENC、BPC
4. **使用 AEC2**：硬件加速更快
